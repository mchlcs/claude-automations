Explain the provided code or concept for someone learning programming.

> **Note:** User input and output can be in PT-BR. Prompts are English for portability.

## Explanation Structure

### In One Sentence
A real-world analogy. No technical jargon.

### What's Happening (line by line)
Walk through the code explaining each important part as if the reader has never seen it.
Use inline comments when helpful:
```java
// What this line does and why
code here
```

### Why Was It Written This Way?
What problem does this code solve? Why wasn't it done differently?

### Common Confusion
What do beginners commonly get wrong or misunderstand here?

### Test Your Understanding
A simple question for the reader to verify they understood.

Tone: colleague explaining to colleague, not teacher to student. No filler.
